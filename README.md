ft\_printf
==========

Description
-----------

The **ft\_printf** project is a fundamental challenge in the 42 curriculum that requires recoding the libc printf() function. The primary goal is to learn about **variadic functions** in C and how to handle a variable number of arguments. This project is of moderate difficulty and focuses on creating well-structured, extensible code that mimics the behavior of the original system function.

Instructions
------------

### Compilation

The project must be compiled into a library named libftprintf.a. You must use the ar command for library creation; libtool is strictly forbidden.

    

### Rules

*   **Flags:** Compilation uses cc with -Wall -Wextra -Werror.
    
*   **Makefile Rules:** Must include $(NAME), all, clean, fclean, and re.
    
*   **Memory:** All heap-allocated memory must be properly freed to prevent leaks.
    

Features
--------

The function handles the following conversions:

**SpecifierDescription**%c

Prints a single character.

%s

Prints a string.

%p

Prints a void pointer in hexadecimal format.

%d / %i

Prints a decimal (base 10) number.

%u

Prints an unsigned decimal (base 10) number.

%x

Prints a number in lowercase hexadecimal (base 16).

%X

Prints a number in uppercase hexadecimal (base 16).

%%

Prints a percent sign.

Algorithm and Data Structures
-----------------------------

To implement ft\_printf, I utilized a **dispatch mechanism** based on the character following the % sign.

*   **Variadic Arguments:** The implementation relies on the stdarg.h macros: va\_start, va\_arg, and va\_end to navigate the variable argument list.
    
*   **Parsing Logic:** The function iterates through the format string. When a % is encountered, a sub-function parses the specifier and routes it to the appropriate conversion handler.
    
*   **Data Structure:** No complex data structures were required for the mandatory part, but a modular architecture was chosen to ensure extensibility for future additions to the library.
